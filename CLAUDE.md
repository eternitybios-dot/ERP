## プロジェクト概要

強迫性障害（OCD）のERP（曝露反応妨害法）治療を補助するための、スマホ用PWA（Progressive Web App）。
「不安を下げるアプリ」ではなく、**強迫行為をしなかった行動の積み重ねを加点式で記録するアプリ**。

GitHub Pages で公開 → iPhoneのSafariで「ホーム画面に追加」→ アプリ感覚で使える形。

## iOS固有の落とし穴（変更禁止事項）

- `apple-mobile-web-app-status-bar-style` は **`black-translucent` から変更しないこと**。
  `default` に変えるとホーム画面アプリでレイアウトビューポートが縮み、ボトムナビが画面下端から浮くバグが起きる（実機で確認済み）。ステータスバーの文字色は `theme-color`（ライト系）から自動で暗色になるので問題ない。
- セーフエリアは CSS の `--safe-t` / `--safe-b`（`env(safe-area-inset-*)`）で自前管理している。
- **iOS 26 WebKitバグ**：スタンドアロン（ホーム画面アプリ）でレイアウトビューポートが画面下端より手前で終わり、`position: fixed; bottom: 0` の下に帯が出る（Safariでは正常）。対策として `html` のキャンバス背景をナビと同じ白 `#ffffff` にして帯をナビの続きに見せている（`body` は `--bg`）。OS側の修正後も無害。この `html`/`body` の背景分離を崩さないこと。

## アプリの画面構成（ボトムナビ5タブ）

| 画面ID | 内容 |
|---|---|
| `#view-home` | 「いま波が来てる」ボタン・今日の点数・**今日の練習（計画的曝露の提案）**・統計・ワンタップ記録5つ＋穏やかな日・リマインダー・使い方再表示 |
| `#view-erp` | 強迫（ERP）記録フォーム（`form-compulsion`） |
| `#view-hrt` | クセ・チック（HRT）記録フォーム（`form-tic`）。動き選択で拮抗反応ガイド表示 |
| `#view-history` | 日付別の記録一覧（強迫＝teal / クセ＝lavのタグで色分け）・バックアップ／復元 |
| `#view-graph` | がんばりの記録（累計ブレイブ・マイルストーン・成長カーブ・週比較・練習の足あと・ブレイブ回数グラフ・発見ノート・月1セルフチェック） |

ビュー切り替えは `app.js` の `showView(name)`。`erp`/`hrt` 表示時に `recordType` を切り替える（ERPとHRTは独立タブ）。オーバーレイ（記録完了フィードバック・呼吸ガイド）は `.overlay.show` で表示。

### 2つの治療トラック（重要）

- **強迫（compulsion / ERP）**: 曝露反応妨害。強迫行為・確認・打ち消しに「反応しない」練習を加点。
- **クセ・チック（tic / HRT）**: ハビット・リバーサル。目パチパチ・鼻息など反復運動の**前駆衝動に気づき、拮抗反応（両立しない別の動き）に置き換える**練習を加点。動きを選ぶと `CR_GUIDES`（app.js）の拮抗反応ガイドを表示。
- 呼吸ガイド（`openBreathing()`）は「衝動の波をやり過ごす」アニメ。鼻呼吸の拮抗反応も兼ねる。

## 加点ルール（重要・設計の核心）

**減点なし。できた行動だけ加点。** ロジックは `scoring.js`（`record.type` で分岐）。

**強迫（ERP）系**
| 条件 | 点数 |
|---|---|
| 記録した（常時） | +1 |
| 強迫行為をしなかった / 少し・遅らせた | +15 / +5 |
| 波をやり過ごせた時間（最高ランクのみ） | 10秒+2 / 1分+5 / 3分+8 / 10分+12 / 放置+15 |
| 予想より耐えられた（期待違反＝抑制学習の核心） | +5 |
| 不快なまま生活に戻れた | +20 |
| 確認・安心探しをしなかった | +20 |
| 打ち消しイメージをしなかった | +15 |

**クセ・チック（HRT）系**
| 条件 | 点数 |
|---|---|
| 記録した（常時） | +1 |
| 前駆衝動に気づけた | +5 |
| 拮抗反応ができた / 少しできた | +15 / +7 |
| 衝動の波をやり過ごせた | +10 |

**穏やかな日（`type: 'calm'`）**: +1。症状が少なかった日をワンタップで記録し、「記録なし日」と区別する（欠測データ対策）。

**計画練習（`planned: true`）**: 記録+1 ＋ 取り組み+10 ＋ 期待違反なら+5（最大16）。
**加点は「取り組んだこと」のみに随伴させる**——タイマーの長さ・途中で強迫行為が出たかで増減させない（結果に随伴させると結果をコントロールしようとする＝強迫の構造になるため）。

点数の正確さにこだわらせない設計にすること。点数付けそのものが強迫化しないよう注意。

## データモデル

**タイムゾーン方針（重要）**: `timestamp` はISO(UTC)で保存。表示・日別集計・「今日」判定はすべて `Storage.localDateStr()` / `localTimeStr()` で**端末ローカル時間**に変換して行う。`timestamp.split('T')` で直接日付や時刻を取り出してはいけない（UTCのままズレる）。

### `localStorage` キー: `ocd_records`

`type` で3種類。旧データ（`type` なし）は `storage.js` で `'compulsion'` 扱い。不快度等は**0〜10スケール**（旧0〜100データは起動時に自動換算）。

```jsonc
// 強迫（ERP）
{ "id": "…_a3f", "timestamp": "…", "type": "compulsion",
  "discomfortLevel": 8, "urgeLevel": 9, "situation": "仕事",
  "triggers": ["汚染感"], "reaction": "しなかった",
  "enduranceTime": "10分", "expectancy": "予想より耐えられた",
  "bonuses": ["不快なまま生活に戻れた"], "memo": "", "score": 53 }

// クセ・チック（HRT）
{ "id": "…_b7c", "timestamp": "…", "type": "tic",
  "movement": "鼻息を強く吐く", "urgeLevel": 7,
  "awareness": true, "competing": "できた", "urgePassed": true, "memo": "", "score": 31 }

// 穏やかな日
{ "id": "…_c1d", "timestamp": "…", "type": "calm", "memo": "", "score": 1 }

// 数の契約をしない（回数・数合わせの衝動。数/時間/達成度は一切持たない）
{ "id": "…_e2f", "timestamp": "…", "type": "contract",
  "outcome": "dropped", "memo": "", "score": 16 }  // dropped=16 / stopped=13（両方とも勝ち）

// 計画練習（今日の練習から実施。typeは compulsion / tic のどちらか）
{ "id": "…_d9e", "timestamp": "…", "type": "compulsion", "planned": true,
  "practiceName": "ドアノブに触れて手を洗わず次へ", "difficulty": "ふつう",
  "predicted": 4, "expectancy": "予想より耐えられた",
  "insight": "思ったよりすぐ波が引いた", "practiceSeconds": 95, "memo": "", "score": 16 }
```

### `localStorage` キー: `ocd_practice_menu` / `ocd_practice_today`（今日の練習）

```jsonc
// ocd_practice_menu: 練習メニュー（テンプレ＋自作）
[{ "id": "…", "name": "…", "difficulty": "やさしい|ふつう|チャレンジ", "type": "compulsion|tic" }]

// ocd_practice_today: 今日のカードUI状態（日付が変わると自動リセット。パス履歴は残さない＝罪悪感のデータ化防止）
{ "date": "2026-07-02", "status": "pending|passed|done", "swaps": 0, "itemName": "…" }
```

### `localStorage` キー: `ocd_checkins`（月1セルフチェック）

```jsonc
[{ "date": "2026-07-02", "answers": [1, 2, 0, 3], "total": 6 }]  // 各0〜4、合計0〜16
```

### `localStorage` キー: `ocd_reminder` / `ocd_reminder_last_shown`（リマインダー）

```jsonc
{ "enabled": false, "time": "20:00" }  // ocd_reminder（デフォルトOFF）
"2026-07-02"                            // ocd_reminder_last_shown（1日1回制御）
"1"                                     // ocd_onboarded（初回オンボーディング済みフラグ）
```

### 日本語の改行（重要）

ブラウザは日本語をどこでも改行できると解釈するため、放置すると「自／然」のように単語の途中で割れる。対策として **BudouX**（`js/budoux.js`、vendored・約29KB）で文節境界にゼロ幅スペースを挿入し、CSS `word-break: keep-all` でその境界（と句読点）でだけ改行させている。

- `app.js` の `decorateJp()` が `window.JpWrap.decorate(document.body)` を呼ぶ（テキストノードのみ書き換え・冪等・イベント非破壊）。ビュー切替・各オーバーレイ表示・動的描画（履歴/練習カード/メニュー/ガイド等）の後に呼んでいる
- 手動 `<br>` は原則使わない（画面幅で崩れる）。文中改行はBudouXに任せる。合言葉など意図的な短い行のみ `<br>` 可
- `budoux.js` は `bundle: budoux` を esbuild で IIFE 化したもの。再生成する場合も `window.JpWrap.decorate` を公開する形を維持

### UI文言の方針

- ボトムナビは平易語（「強迫」「クセ」）。専門用語（ERP/HRT/拮抗反応）は画面内の括弧併記で学習機会として残す
- 記録完了画面の「取り消す」は誤タップ救済専用。取り消し後に再入力を促す導線は作らない（完璧な記録を求めさせない）
- 呼吸ガイドから記録できるのは「やり過ごした行動」。不安が下がったかどうかを問わない

### バックアップ形式（v2）

`{ app, version: 2, exportedAt, records, checkins }` の封筒形式。旧形式（recordsの生配列）も復元可。

## 設計上の注意点

- **点数付けの強迫化を防ぐ**: 減点・失敗表示・警告・ランキングは入れない
- **「不安が下がったか」ではなく「反応しなかった／置き換えたか」で点数をつける**
  - ERPは抑制学習モデルに基づく。慣れ（不安が下がること）を目的化しない＝「下がるまで待つ」は安心探しになるので避ける。衝動は波として「やり過ごし、生活に戻る」言い回しに統一。
  - **「ガマン」「我慢」という言葉は使わない**（抑え込む＝思考抑制のリバウンドを招く枠組みになる）。「反応しなかった」「やり過ごした」「しないでいられた」を使う。
  - チック／反復運動はHRT/CBIT（気づき訓練＋拮抗反応）が最もエビデンスのある介入。動きを止めるのではなく置き換える設計にする。拮抗反応は約1分保持（CRタイマーで支援）。
  - **期待違反（予想より耐えられた）が抑制学習の最重要シグナル**。ERPフォームの `expectancy` で記録し、+5と発見メッセージで強化する。
- **連続日数は「やさしい判定」**（1日空きはセーフ・2日連続空きで区切り・今日未記録でも切れない）。ストリークの喪失が罰にならないようにする。主役は減らない「練習した日」の合計。
- **「日同士を比べる表示」はしない**：日別合計点グラフ・「最高の1日」は廃止済み。記録回数は暇な日ほど増える（volume交絡）ため、日単位の指標は「記録したか/してないかの2値ドット（練習の足あと）」と減らない累計のみ。**日別平均点は禁止**——穏やかな日+1や「気づけた」+6を記録すると平均が下がり、小さな勝ちの記録が抑制されるため（点数は日のパフォーマンス指標ではなく、1件ごとの即時強化子）。
- 記録前後に「正確じゃなくていい」「不快感が残っていても成功」のメッセージを表示する
- メモ欄は最大100文字に制限（反芻防止）。前回引き継ぎ（`prefillFromLatest`）でもメモ・expectancyは引き継がない
- 入力負荷を最小化：ホームのワンタップ記録（`quickLog()`）を主動線にする
- **月1セルフチェックは任意・低頻度固定**。催促はがんばりタブ内のみ（プッシュ・バッジ禁止）。点数は付けない（測定の強迫化防止）
- **リマインダーはオプトイン（デフォルトOFF）・1日1回だけ・責めない文言**。サーバーレスのためiOSでは「アプリを閉じていても届くWeb Push」は不可能 → アプリ内通知（`showNotification`）＋確実な手段として毎日繰り返しの `.ics` カレンダー通知生成の2段構え。通知を増やす変更（回数追加・未記録の催促など）はしない
- 履歴のHTML生成では、記録由来の文字列を必ず `esc()` に通す（バックアップ復元経由の注入対策）。削除ボタンは `data-id` ＋イベントデリゲーション（onclick属性に値を埋め込まない）

### 今日の練習（計画的曝露）の設計原則（世界的権威レビューの5条件）

1. **加点は行動（取り組んだこと）に随伴、結果に随伴させない**。タイマー長・強迫行為の有無で+10を増減しない
2. **発見ノートは安心探しの道具にしない**。再表示は練習開始前の文脈＋ホームで週1程度の低頻度のみ。「お守りではない」注意文を常設
3. **回避ドリフト対策**：直近3回連続「やさしい」完了→次の提案は「ふつう」1回（`Scoring.suggestTier`、決め打ちルール・テスト済み）。入れ替えは常に自由
4. **チックの衝動曝露は短く始める**：ヒント文で「まずは30秒からでOK」
5. **実施率（%）は表示しない**。増えるだけの回数のみ（完璧主義の燃料にしない）
- パスは記録しない（`ocd_practice_today` は当日のUI状態のみ）。パス時の催促・未実施表示は入れない

### 数の契約をしない練習（`type: 'contract'`）の設計原則

回数・数合わせの衝動（例：「N回瞬きすれば安心」）＝チックに併存する数完了OCDへの介入。潰す対象は動きではなく「決まった数を完了すれば安心できる、という脳との契約」。

- **数・時間・達成度・不安の増減を一切扱わない**。何回やったか、ちゃんとできたか、不安が下がったかは記録も表示もしない（測定自体が数完了の儀式を再現するため）
- **タイマー・目標秒数・完了ボタンを置かない**（HRTの1分キープは流用禁止＝「完了」が新たな数の契約になる）。力を抜いて“今すぐ”戻る、のみ
- 結果は2択で**両方とも肯定・加点**：`dropped`（契約せず戻れた）/ `stopped`（出たがやり直さず終了できた）。`stopped` を失敗扱いしない（やり直さなかったこと自体が治療の勝ち）
- 合言葉は**固定テキスト**（編集・推敲させない＝推敲が儀式化する）。「お守りではなく契約から降りる合図」の注意文を常設
- 心理教育は初回1回のみ（`ocd_contract_intro`）。使用回数を賞賛するバッジ・連続記録は作らない
