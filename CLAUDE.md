## プロジェクト概要

強迫性障害（OCD）のERP（曝露反応妨害法）治療を補助するための、スマホ用PWA（Progressive Web App）。
「不安を下げるアプリ」ではなく、**強迫行為をしなかった行動の積み重ねを加点式で記録するアプリ**。

GitHub Pages で公開 → iPhoneのSafariで「ホーム画面に追加」→ アプリ感覚で使える形。

## アプリの4画面構成

| 画面ID | 内容 |
|---|---|
| `#view-home` | 今日の点数・統計・ワンタップ記録3つ・呼吸ガイド入口 |
| `#view-record` | 記録フォーム。「強迫（ERP）」「クセ・チック（HRT）」を `setRecordType()` で切替 |
| `#view-history` | 日付別の記録一覧（強迫＝teal / クセ＝lavのタグで色分け）・バックアップ／復元 |
| `#view-graph` | 直近7日間の棒グラフ2本（日別点／反応しなかった・置き換えた回数） |

ビュー切り替えは `app.js` の `showView(name)`。オーバーレイ（記録完了フィードバック・呼吸ガイド）は `.overlay.show` で表示。

### 2つの治療トラック（重要）

- **強迫（compulsion / ERP）**: 曝露反応妨害。強迫行為・確認・打ち消しに「反応しない」練習を加点。
- **クセ・チック（tic / HRT）**: ハビット・リバーサル。目パチパチ・鼻息など反復運動の**前駆衝動に気づき、拮抗反応（両立しない別の動き）に置き換える**練習を加点。動きを選ぶと `CR_GUIDES`（app.js）の拮抗反応ガイドを表示。
- 呼吸ガイド（`openBreathing()`）は「衝動の波をやり過ごす」アニメ。鼻呼吸の拮抗反応も兼ねる。

## 加点ルール（重要・設計の核心）

**減点なし。できた行動だけ加点。** ロジックは `scoring.js`（`record.type` で分岐）。

**強迫（ERP）系**
| 条件 | 点数 |
|---|---|
| 記録した（常時） | +1 |
| 強迫行為をしなかった / 少し・遅らせた | +15 / +5 |
| 波をやり過ごせた時間（最高ランクのみ） | 10秒+2 / 1分+5 / 3分+8 / 10分+12 / 放置+15 |
| 不快なまま生活に戻れた | +20 |
| 確認・安心探しをしなかった | +20 |
| 打ち消しイメージをしなかった | +15 |

**クセ・チック（HRT）系**
| 条件 | 点数 |
|---|---|
| 記録した（常時） | +1 |
| 前駆衝動に気づけた | +5 |
| 拮抗反応ができた / 少しできた | +15 / +7 |
| 衝動の波をやり過ごせた | +10 |

点数の正確さにこだわらせない設計にすること。点数付けそのものが強迫化しないよう注意。

## データモデル（`localStorage` キー: `ocd_records`）

`type` で2種類。旧データ（`type` なし）は `storage.js` で `'compulsion'` 扱い。不快度等は**0〜10スケール**（旧データは0〜100が混在しうる）。

```jsonc
// 強迫（ERP）
{ "id": "…_a3f", "timestamp": "…", "type": "compulsion",
  "discomfortLevel": 8, "urgeLevel": 9, "situation": "仕事",
  "triggers": ["汚染感"], "reaction": "しなかった",
  "enduranceTime": "10分", "bonuses": ["不快なまま生活に戻れた"], "memo": "", "score": 48 }

// クセ・チック（HRT）
{ "id": "…_b7c", "timestamp": "…", "type": "tic",
  "movement": "鼻息を強く吐く", "urgeLevel": 7,
  "awareness": true, "competing": "できた", "urgePassed": true, "memo": "", "score": 31 }
```

## ローカルでの確認方法

ビルドは不要。ブラウザで直接開けばOK。

```bash
# 方法1: Pythonの簡易サーバー（ポート8080で起動）
python3 -m http.server 8080
# → http://localhost:8080 で確認

# 方法2: VS Codeの Live Server 拡張を使う
```

PWA（Service Workerなど）の動作確認は `http://localhost` か `https://` 環境が必要。
`file://` で開くと Service Worker が動かないので注意。

## 設計上の注意点

- **点数付けの強迫化を防ぐ**: 減点・失敗表示・警告・ランキングは入れない
- **「不安が下がったか」ではなく「反応しなかった／置き換えたか」で点数をつける**
  - ERPは抑制学習モデルに基づく。慣れ（不安が下がること）を目的化しない＝「下がるまで待つ」は安心探しになるので避ける。衝動は波として「やり過ごし、生活に戻る」言い回しに統一。
  - チック／反復運動はHRT/CBIT（気づき訓練＋拮抗反応）が最もエビデンスのある介入。動きを止めるのではなく置き換える設計にする。
- 記録前後に「正確じゃなくていい」「不快感が残っていても成功」のメッセージを表示する
- メモ欄は最大100文字に制限（反芻防止）。前回引き継ぎ（`prefillFromLatest`）でもメモは引き継がない
- 入力負荷を最小化：ホームのワンタップ記録（`quickLog()`）を主動線にする
